웹기초-js

3가지 이상의 기능/요소 등의 처리를 반복 수행할 경우에는 매번 하나씩 만드는 것이 아닌, 반복문을 사용하여 처리

  • whlie : 여러개의 반복 수행을 처리하는 문법 - whlie(){} 다른 반복문에 비해 속도가 느리기도하며, 특정조건 수행 후 반복을 끝내는 기능을 미작성하는 경우가 있어, 권장하지 않는 기능
  • do-while : whlie의 확장형으로 조건을 파악했을 때 기본반복문은 처음부터 수행하지 않을 수 있지만, do-whlie은 무조건 최소 1번은 수행 - do{} whlie()
  • for(v) : 가장 많이 사용되는 반복문의 기본형태이며, 최초의 값/ 조건/ 이후 수행처리에 대한 내용을 한군데 모두 작성하여, 반복문의 수행 속도를 개선하게 된 처리방법 - for( ; ; ){}
  • for-in : for의 변형된 형태로 배열에서 많이 사용하는 형태로 알려져있지만, 반대로 배열에서는 지양/ 객체({})에서 사용되는 기법 - for( in ){}
  • forEach(v) : for의 변형된 형태, 배열의 반복으로 사용되는 기법으로 배열요소를 각각 수행한다는 의미 - [].forEach(function(data, index){} )
